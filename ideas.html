Ideas for Developers

Potential improvements and features for the 1/f Voice Analyzer.
Pick one up and send a pull request!

Analysis & Signal Processing
📐

True Spectral Slope via Log–Log Regression

Replace the current DFT-loop approach with a proper FFT (e.g. using the Web Audio API's AnalyserNode) and fit the power spectral density in log–log space. A 1/f signal should yield a slope ≈ −1. Report the slope and R² clearly.

🔇

Smarter Silence / Noise-Floor Detection

The current 0.02 RMS threshold is a hard-coded guess. Compute the noise floor from the first silent frames and set the threshold dynamically (e.g. noise floor + 10 dB). This reduces false positives on quiet microphones.

Pitch & Jitter / Shimmer Metrics

Add optional voice-quality metrics (fundamental frequency F0, jitter, shimmer) alongside the 1/f score. These are standard in clinical voice research and give users more nuanced feedback.

Real-Time Waveform / Spectrum Preview

Show a live waveform or spectrum canvas during recording so users know the microphone is picking up their voice. A simple AnalyserNode drawing loop is enough.

Engineering & Performance
⚡

Move Analysis to a Web Worker

The current analysis runs on the main thread and can freeze the UI for long recordings. Moving the DFT loop into a Worker keeps the page responsive and is a clean architectural improvement.

PerformanceWeb API
Medium difficulty
🧪

Unit Tests for the Analysis Logic

Add a minimal test suite (e.g. with Vitest or plain assert) that feeds synthetic 1/f noise into analyze() and asserts the slope is near −1. This would catch regressions when the algorithm is changed.

PWA / Offline Support

Add a Service Worker and a manifest.json so the app can be installed on mobile home screens and work without a network connection. All analysis is already local, so this is purely a shell addition.
